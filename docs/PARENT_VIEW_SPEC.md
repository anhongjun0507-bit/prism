## 미결정 사항

### Q1. 학부모 접근 방식

| 옵션 | 설명 | 장점 | 단점 |
|---|---|---|---|
| **A. 학부모 별도 가입** | 학부모용 회원 가입 → 학생 계정과 연결 | 권한 분리, 학부모 전용 알림 채널 가능 | 가입 마찰, 본인인증 부담, 페르소나 P1/P3에는 진입장벽 |
| **B. 학생 계정 공유** | 학부모가 학생 ID/PW로 로그인 | 구현 0, 즉시 가능 | 보안 위험, 학생 작성 메모/에세이 노출, 분석 추적 불가 |
| **C. 학생 발급 view-only 링크** | 학생이 "학부모용 링크" 생성, 학부모는 로그인 없이 접근 | 가입 부담 0, 학생이 통제 가능, 카톡 공유 친화 | 링크 유출 시 노출, 데이터 권한 세분화 필요 |

**현 인프라**: parent email 필드 없음, parent account 없음. 옵션 A는 schema 추가 + 본인인증 필요(2주 견적 초과 위험).

### Q2. 학부모 뷰의 주 목적

| 옵션 | 설명 | 영향 받는 페이지 |
|---|---|---|
| **A. 결제 설득 (마케팅)** | `/sample-report` 확장. 가상 데이터로 "이런 게 옵니다" | `/sample-report` |
| **B. 진척 모니터링 (정기 리포트)** | `/parent-report` 재설계. 실제 학생 데이터 학부모 친화 변환 | `/parent-report` |
| **C. 소통 도구 (양방향)** | 학생-학부모 코멘트, 격려 메시지, 알림 | 새 페이지 군 |

평가자 원문은 "/parent-report 확장. 큰 글자, 단순 네비, 요약 중심"으로 옵션 B에 가장 가까움.

### Q3. 플랜 분기

| 옵션 | 설명 | 결제 전환율 영향 (추정) |
|---|---|---|
| **A. Elite 전용 (현재 그대로)** | weekly 리포트만 Elite, basic은 Pro | 현재 전환율 유지 |
| **B. Pro부터 부분 노출, Elite는 풀** | 부분 정보로 Elite 매력 강조 | 중간 — Pro→Elite 전환 ↑ 가능 |
| **C. Free도 학부모 뷰 일부** | 월 1회 무료 → 결제 funnel 확장 | 상단 funnel ↑, 다운스트림 전환은 측정 필요 |

현재 `parentReportType` enum (`none`/`sample`/`basic`/`weekly`)으로 옵션 A 구조. 옵션 B/C는 기존 enum 유지하면서 페이지 내 부분 잠금/일부 모듈 공개로 구현 가능.
